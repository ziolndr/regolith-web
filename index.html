Rails Microservices Made Simple

Setting up Rails microservices means hours of manual configuration: generating individual Rails APIs, setting up Docker containers, configuring orchestration, and managing service discovery.

Regolith automates this entire workflow. Generate production-ready Rails services, fully containerized and orchestrated, in seconds instead of hours.

How Regolith Works

Three commands to production-ready microservices architecture.

01

Initialize Platform

Creates project structure with Docker orchestration, PostgreSQL configuration, and service registry.

regolith new myapp
02

Generate Services

Scaffolds complete Rails API apps with Docker containers, CORS configuration, and automatic port assignment.

regolith generate service users
03

Deploy & Scale

Starts all services with Docker Compose. Each service scales independently with production PostgreSQL.

regolith server

Built for Production

Complete Rails APIs

Each service is a full Rails 7 API application with PostgreSQL, CORS configuration, and production-ready structure.

rails new --api with custom optimizations

Docker Orchestration

Automatic Docker containerization with optimized Ruby 3.1 images, volume mounting, and multi-service networking.

docker-compose.yml generation and management

Service Discovery

Built-in service registry tracks all services with automatic port assignment and environment configuration.

config/regolith.yml service coordination

Development Tools

Rails console access per service, hot-reload development, and integrated debugging across the entire platform.

regolith console <service_name>
